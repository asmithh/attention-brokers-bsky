20260212 2pm:
Added plots.ipynb, which plots per-day per-capita follower accumulation rates for 
both non-followers of the attention broker as well as the attention broker's followers.
plots.ipynb also contains functionality for making the CSVs that are fed into the 
differences-in-differences analysis. 

Also added diff_in_diff.R, in consultation with Fiona. This contains code for estimating
an overall post-treatment effect of exposure to a repost, as well as an event study
for estimating the effect of a repost on per-day per-capita following rate. For the data
we have (followers + non-followers' following rates over 28 days for each reposted account;
very spread out over time; lacking measurements over the entire time period studied; no 
never-treated units), it makes more sense to treat followers' per-day per-capita following
rates as the outcome variable in one DiD analysis (where all units are treated), and
then treat non-followers' per-day per-capita following rates as the outcome variable in
another DiD analysis (where, again, we count all units as treated). Then we can compare
the effect sizes for followers and non-followers to say that being an attention broker's 
follow causes an increase in following rate on (e.g.) day 0.

So far, the Bluesky data is a mixed bag. Aaron Rupar looks like an attention broker, but

Oswalt -- look very different, but they're different from Aaron Rupar in very similar ways.
I don't think the other 3 are attention brokers though.

20260216 10am:
Realized that the code for counting follower and non-follower populations was
incorrectly assigning followers to the attention broker that should be
non-followers because I had done the conditional assignment of ab_follower
such that accounts who had followed the reposted account post-repost would be
assigned as followers if the date at which they followed the reposted account
was greater than the day of the repost (which of course it would be). I have
corrected this, and now accounts that followed the reposted account *before*
the repost are assigned as followers if they followed the reposted account
*after* having followed the attention broker. Accounts that followed the
reposted account *after* the repost are assigned as followers if they followed
the attention broker before they followed the reposted account. This bug was
not replicated in the parsing code that makes the DiD CSVs because I treated
the followers and non-followers separately and did not introduce any incorrect
conditionals. Nonetheless, I'm also setting the null values for the
created_at_from_ab column (i.e. when the attention broker was followed) equal
to the day of the repost + 365 * 5 days. This ensures, beyond the specter of a
doubt, that people who never followed the attention broker will not be
assigned as followers because of some zero overflow/underflow error with
subtraction in polars. Not saying it won't happen for some other reason, but
at least we're covering that base.
